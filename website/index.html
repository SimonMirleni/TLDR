Workflow

Save now. Read later. Digest when ready.

ReadLater Digest is intentionally on demand. Nothing is scheduled, selected, or reshuffled for you.

01

Capture the page

Save the current tab text from the popup, or import unread Chrome Reading List items.

02

Keep a pending queue

Saved resources stay pending in oldest-first order until they are included in a digest.

03

Send one newsletter

The backend summarizes the next N resources and emails the result to your Google address.

What it can do

Current-page saving

Capture the active tab text directly from the extension popup.

Reading List import

Bring unread Chrome Reading List entries into the same digest queue.

New-tab previews

See pending reads as quick preview cards whenever you open a new tab.

Custom digest controls

Choose how many resources go into each newsletter and set the summary tone prompt.

FIFO by design

The send flow selects the oldest pending resources first, with no manual picking.

Consumed after send

Sent resources leave the pending queue so the next digest moves forward.

Email output

A calm inbox version of your backlog.

Each digest keeps the original links, adds short summaries, and highlights the main points.
